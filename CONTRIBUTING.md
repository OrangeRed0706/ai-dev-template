# Contributing

歡迎貢獻！

## 如何貢獻

1. **Bug 修正 / 小改動** → 直接開 PR
2. **新功能 / 架構變更** → 先開 Issue 或 Discussion 討論
3. **問題** → 開 Issue

## PR 前檢查

- 本地測試過
- 跑過完整檢查（build + lint + test）
- 保持 PR 專注（一個 PR 一件事）
- 描述清楚做了什麼、為什麼

## AI 輔助 PR

用 AI 工具（Claude, Codex, Cursor 等）寫的 PR 完全歡迎！請在 PR 裡說明：

- [ ] 標註是 AI 輔助
- [ ] 說明測試程度（未測試 / 輕度測試 / 完整測試）
- [ ] 確認你理解程式碼在做什麼

## Commit Message

使用 Conventional Commits：

```
feat: 新功能
fix: 修 bug
docs: 文件
refactor: 重構
test: 測試
chore: 雜項
```

## Code Style

- 遵循專案既有風格
- 跑 lint/format 再 commit
- 加註解解釋複雜邏輯
